Web review: reminders

There are a few things that bear reminding:

  • file and folder names cannot contain:
  • capital letters
  • spaces
  • special characters:
! @ # $ % ^ & * ( ) + { } [ ] : ; " < > ? |
  • acceptable non-alphanumeric characters: . _ - ~
  • default page: index.htm, index.html or index.shtml (or whatever your server default is)

Homework review

As this is a web development course, let’s talk a bit about coding patterns & behaviour.

Comments

Comments in your code are great! For you, anyone you work with or anyone looking at your site, both today and especially in the future.

  • Code should tell the story, comments are just additional notes. Be brief!
  • Use tags to help you out
  • Minimize the amount you comment on
  • Opening tags unnecessary

Browser Tools

Lots of options

These days there are lots of browser-based tools available for web developers. Initially, Mozilla (Firefox) built in some basic tools and their plug-ins centered around web development.

Now, Webkit (Chrome, Safari) and Mozilla (Firefox) have excellent tools built-in. Some need to be “turned on” but others are built in by default.

The tools are enhanced by their organizations and are ever-increasing in capability and what they can do.

Do not underestimate how powerful these tools can be! I can guarantee these tools will get you out of trouble, or solve a problem you didn’t even know you had.

What kinds of things do these tools help you with?

  • Checking validity of your code
  • See errors in rendering
  • Use the console for Javascript output (remember to code your error throwing!)
  • Slow loading resources
  • Large resources
  • External bottlenecks
  • Rendering issues
  • … much more!

While we will look at Chrome and Safari, the focus will be Firefox in COMP1950 so that we are all working from the same platform.

Remember: to check in all browsers when taking a look at your site for production.

Week Two Homework

Create Two Realtor Pages

You will create the same page, but twice: One in XHTML Strict and the other using HTML5

Details

You will create a web page for a Realtor (fictional or real), which we will use as the course goes on.

Required components

Take a look at existing Realtor sites and take a look at some of the standard elements in each.

At a minimum, each version (XHTML and HTML5) should include:

  • Realtor photo
  • Company information (Company name, Logo)
  • Contact information (phone, email, address)
  • Where in the lower mainland they work
  • At least 2 current listings
  • At least 2 sold listings
  • Short Realtor biography

The page does not have to be big, think of it as a summary page. We will build listings and more detail later.

Design

  • The focus of your page should be the inclusion of information rather than the presentation.
  • Use CSS to create the presentation layer, fonts and measurements don’t have to be exact.
  • Use any layout scheme you would like (tables, CSS, etc)

Versions

There will be two versions of the page you create. They should both contain the same information but built with two different web technologies:

  • XHTML Strict - Be sure to review the requirements for an XHTML Strict page
  • HTML5 - Review what we discussed to day about HTML5 and be sure to include tags from HTML5
